第4章 方法

4.1 方法的定义

在程序之中，很多情况下是需要重复执行一些代码的。在有一些书中也称方法为函数，它们是相同的概念，在Java中称之为方法（method）。需要注意的是，本次进行方法定义时有个前提：方法在主类中定义并且由主方法直接调用。因为方法定义限制复杂，在此先不涉及跨文件调用方法。

对于返回值而言，就可以使用 Java 中定义的数据类型了（基本数据类型，引用型数据类型），在方法中可以进行返回数据使用 return 来描述，return 所返回的数据类型与方法定义的返回值数据类型相同，如果不返回数据则该方法可以使用 void 进行声明。

方法命名要求：首字母小写，之后单词首字母大写，如”printInfo()”，与变量名命名规则相同。

4.3 方法递归调用

方法递归调用指的是方法自己调用自己的情况，利用递归调用可以解决一些重复且麻烦的问题。在进行方法递归调用的时候，一般要考虑如下几点问题：

  • 一定要设置方法递归的结束条件。
  • 每次调用的时候都需要根据需要改变传递参数的内容。

实例：用方法递归调用实现 1 -100 累加：

第 1 次执行 sum() ：由主方法调用，return 100 + sum(99) 。
第 2 次执行sum() ：由 sum() 调用，return 99 + sum(98) 。
…
第 99 次执行sum() ：由 sum() 调用，return 2 + sum(1) 。
第 100 次执行sum() ：由 sum() 调用，return 1 。
返回 第 100 次调用： sum(1) return 1 。
返回 第 99 次调用： sum(2) return 2 + 1 = 3。
…
返回 第 2 次调用： sum(99) return 99 + 98+ … + 2 + 1 = 4950。
返回 第 1 次调用 给主方法： sum(100) return 100 + 99 + 98+ … + 2 + 1 = 5050 。

递归操作虽然可以简化调用，但是在实际开发之中，很少出现递归操作。大部分情况下考虑都是些简单的处理逻辑。递归较少使用的另一个原因是递归处理不当会导致会造成内存溢出。

实例：计算 1! + 2! + 3! + …. + 90!

阶乘数值较大，超过 long 型范围，使用 double 储存结果。
