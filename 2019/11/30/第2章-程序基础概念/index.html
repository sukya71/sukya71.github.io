# 第2章 程序概念基础

第2章 程序基础概念

2.1 程序注释

程序注释是项目开发中的一项重要组成技术，合理的注释会让项目开发更加的方便，但很多公司技术管理不到位，很多注释不写的，所以就造成了人员更换时期的痛苦。

编译器在进行程序编译的时候发现有注释的内容将不对此部分进行编译，注释分三类形式。

  • 单行注释：// ……

  • 多行注释：/* ……*/

  • 文档注释：/** ……*/，文档注释里还需要很多的选项，一般建议通过开发工具控制。

在使用开发工具时，还是单行代码比较方便，而对于重要的类和方法都建议使用文档注释进行描述。

2.2 标识符与关键字

任何一个程序都是一个结构的整合体，如类、方法和变量。对于不同的结构有不同的说明，这些说明在程序中被称作标识符，是有着命名要求的，一般由有意义的单词组成，对于标识符的组成在Java之中的定义如下：由字母、数字、下划线“_”、“$”所组成，其中不能使用数字开头，不能使用Java中的保留字（关键字）。

最简单的定义形式：使用英文字母开头，驼峰式命名。大小写有区分。尽量不要使用数字进行命名。对于“$”有特殊的意义，不建议出现在自己编写的代码标识符中。

Java关键字是系统对于一些结构的描述处理，有着特殊的含义，不能用来做标识符使用。对于关键字有以下说明：

  • JDK 1.4 中有 assert 关键字，用于异常处理上；
  • JDK 1.5 中有 enum 关键字，用于枚举定义上；
  • 未使用到的关键字：goto、const；
  • 有一些属于特殊含义的词，不算关键字：true false、null。
  • JDK 1.10 中新加 var 关键字，实现动态变量声明。

2.3 数据类型划分

程序是一套数据处理框架，在整个程序开发的过程之中，所达到的最终目的就是对一些数据的处理过程。既然要进行数据处理，那就一定需要提供对各种类型数据提供定义。

数据类型分为两类：基本数据类型、引用数据类型。

  • 基本数据类型：描述一些具体的数字单元，如 1、1.1；
    • 数值型：
      • 整型：byte、short、int、long； 默认值：0
      • 浮点型：float、double； 默认值：0.0
    • 布尔型： boolean； 默认值：false
    • 字符型：char； 默认值：\u0000
  • 引用数字类型：牵扯内存关系的使用：
    数组、类、接口。 默认值：null

每种数据类型都有保存数据范围，精度。在应用中牵扯到数据类型选择问题，对于以上的类型划分有一些使用参考原则：

  • 描述数字的首选是 int（整数）、double（小数）；
  • 在进行数据传输或者进行文字编码转换使用 byte 类型（二进制处理操作）；
  • 处理中文的时候最方便的操作使用字符型 char 来完成；
  • 描述内存或文件大小、描述表的主键列（自动增长）可以用long。

整型数据一共有4种，按保存范围由小到大为：byte、short、int、long，在Java中任何的整型常量，默认的数据类型都是int型（只要是整数就是int型数据）。Java对于变量也有命名标准：第一个首字母小写，随后的每个单词首字母大写，如：studentName、systemInfo。

2.5 浮点型数据类型

浮点型数据描述的是小数，而在Java中任意的一个小数常量数据类型为double，因此在描述小数时建议直接用double对数据进行定义。

进行自动数据类型转换时都是从小类型向大类型进行转换处理。默认类型为double，也可以定义采用位数较少的float变量，此时从赋值时就要进行强制类型转换。

通过一系列代码分析发现，整型不包含小数，而浮点型包含小数点的。

在计算时一定注意选择的数据类型，它将直接决定小数点的问题。JDK 1.10后Java也提供了var关键字，可以定义动态变量。

利用var关键字定义了num动态变量，赋值 10.2 属于 double 类型，因此 num 的类型就为 double，固定不变，随后赋值100虽然为 int 类型，但是 num 类型为 double，所以自动转型为 double 。虽然 Java 提供了这种动态语法，但是对于动态变量的定义不如其他语言强大（Python、JavaScript），因此不建议使用此种类型定义形式。

2.6 字符型数据类型

字符型数据使用 char 进行定义，在 Java 中使用“‘’”定义的内容就是一位字符。在任何编程语言中，字符都可以与 int 进行互相转换，也就是说字符中描述的内容可以通过int获取其对应的系统编码。

  • 大写字母范围（A-Z）：’A’（65） - ‘Z’（90）；
  • 小写字母范围（a-z）：’a’（97）- ‘z’（122）；
  • 数字范围（0-9）：’0’（48）- ‘9’（57）。

可以借此转换大小写转换处理。到此为止所有操作都与传统C语言的方式是一样的，但是需要注意，在Java 中 char 主要是进行中文的处理，Java 中的 char 类型可以保存中文字符数据。之所以能再 Java 中可以使用 char 进行中文字符数据的保存，是因为 Java 使用 Unicode 这种十六进制字符编码，它的特点是编码包括任意字符内容，便于处理中文字符，使得程序开发更加简单。

2.7 布尔数据类型

布尔是一位数学家的名字，布尔类型数据主要描述的是一种逻辑处理结果，在 Java 中使用 boolean 来进行布尔类型数据的定义。boolean 类型数据取值范围只有两个值：true 和 false。

需要注意，像一些编程语言中由于没有提供布尔类型数据，所以会使用 0 表示 false，或者使用非 0 表示 true。而这样的逻辑在 Java 之中是不存在的。true 与 false 是 Java 中的字面量，不能用作标识符。

2.8 String字符串

在任何语言中都没有提供所谓字符串这种基本数据类型，但是从实际使用上来说，各个编程语言为了方便程序开发也都会提供有字符串的相应描述。在 Java 中使用 String 作为字符串的定义。由于 String 类的存在较为特殊，它可以像普通变量那样采用直接赋值的方式进行字符串的定义，并且要求使用“””进行字符串描述。

在进行字符串变量使用的时候也可以使用“+”来进行字符串的连接处理。因此需要考虑“+”的两种描述：字符串的连接和数字的加法计算。

在 Java 语言中，数据范围大的数据类型在于数据范围小的数据类型进行计算的时候，范围小的数据类型自动转化为范围大的数据类型，若有 String 字符串类型，则所有数据无条件变成字符串类型，此时如果有“+”则它表示字符连接处理操作。

在描述字符串的时候也可以使用转义字符进行一些处理，如：TAB（\t）、“（"）、‘（\‘）、换行（\n）、\（\）。

2.9 运算符简介

所有的程序开发都是数据处理游戏，对于数字的处理一定会有所谓的操作模式，而这些操作模式就称为运算符。如进行加法运算使用的“+”这样的运算符表示。对于不同运算符而言，也是存在优先顺序的关系。

在程序开发中，可以使用很多种的基础运算符，这些运算符也会有各自的优先顺序，不推荐硬记，不明显部分的使用括号标明运算顺序，简单明了。关键问题是，对于程序开发而言，不要编写逻辑很复杂的运算式，对于程序代码而言，已经告别了复杂程序逻辑的时代，更多情况下是希望大家编写简单易懂的代码。

2.10 数学运算符

在 Java 中数学运算都提供了标准的支持，包括四则运算模运算等。Java 为了简化运算与赋值操作，提供了简化运算符，表示参与运算后直接进行赋值操作，如num = num + 1在内存小的时代就不合适了，因此有了简化运算符。

  • 简化数值运算符：“+=”、 “-=”、“*=”、 “/=”、“%=”；
  • 自增与自减运算符：“++”、“—”。

自增自减放在变量前表示先自增自减再进行数字计算，放在变量后表示先进行数字计算再自增自减。

这种代码程序都是当初内存量不大的情况下提供的处理方式，但是现在来看这种计算就变的非常繁琐，尤其是在硬件成本不断降低的今天，这类操作如果不是必须的情况下，就不建议使用。

2.11 关系运算符

关系运算符的主要特征就是进行大小关系的比较处理，包括：大于（>）、小于（<）、大于等于（>=）、小于等于（<=）、不等（!=）、相等（==）。所有关系运算返回的判断结果都是布尔类型的数据。

在关系判断时需要特别注意的是相等判断的问题：在Java里面“=”表示的是赋值运算，而内容相同的比较是“==”。

在进行关系运算时可以针对所有的基本数据类型，如进行字符处理。数据类型之间有自动转型支持，所以在进行判断时字符型 char 会自动转为 int，而后进行数字比较。

2.12 三目运算符

在进行程序开发时三目运算符使用的非常多，合理运用三目运算符可以减少一些逻辑判断的编写。三目是一种所谓的赋值运算处理，它需要设置一个逻辑关系的判断之后才可以进行赋值操作，基本语法如下：

数据类型 变量 = 关系运算 ? 满足时的内容 : 关系不满足时的内容 ;

三目运算本身也可以进行嵌套处理，也就是在赋值语句的位置上可以进一步编写三目运算。

虽然可以允许嵌套处理，但是程序的可读性会变差，根据实际情况确定是否使用三目运算。

2.13 逻辑运算符

逻辑运算符主要是三种： 与（&&、&）、或（||、|)、非（!）。

非的本质是针对操作结果的true变false，false变true。

与操作的特征是进行若干个判断条件的连接，全真则真，一假则假。
或操作的特征是进行若干个判断条件的连接，全假则假，一真则真。

&& 为短路与，当运算中其中一个逻辑判断为false时就不判断之后的逻辑，自动跳出返回false。
同样 || 为短路或，当运算中其中一个逻辑判断为true时就不判断之后的逻辑，自动跳出返回true。

在项目开发中一般使用短路与或（&&、||）。

2.14 位运算符

位运算符可以实现对数值类型二进制数进行处理的位运算，分为逻辑运算（~、&、|、^）、移位运算（>>、<<、>>>）两种。

  • & 按位“与”（位与操作） 、| 按位“或”（位或操作）
  • ^ 异或（相同为0，不同为1）
  • ~ 按位取反，反码
  • << 左移位、>> 右移位
  • >>>无符号右移位

我们需要知道十进制与二进制转换处理方法：数字除2取余，如13：

13/2=6 …… 1
6/2=3 …… 0
3/2=1 …… 1
1/2=0 …… 1

13 : 1101

与计算结果: 5（0101） 或运算结果: 15（1111）

面试题：请解释&、&&、|、||、的区别？

  • & 和 | 两个运算符可以进行位运算与逻辑运算。在进行逻辑运算时所有判断条件都要执行，在进行位运算时只是针对当前的数据进行与，或处理。

  • 在进行逻辑运算时还可以使用 && 和 || 。&& 在进行若干条件判断时，如果前面条件返回了false，则后续所有条件跳过不再判断直接返回 false 计算结果。|| 在进行若干条件判断时，如果前面条件返回了true，则后续所有条件跳过不再判断直接返回 true 计算结果。
