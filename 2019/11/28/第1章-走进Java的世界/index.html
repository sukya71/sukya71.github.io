# 第1章 走进Java的世界

第1章 走进Java的世界

1.1 Java发展历史

Java可以实现任何需求，有些时候Java不擅长的部分会被其他语言所替代。国内而言Java使用广泛的原因是有很多的大户都在使用Java实现其各自的核心业务。

1991年 GREEN项目，智能家居，SUN公司打算竞标此项目，考虑到C++语言开发的复杂性，于是基于C++语言开发出了一套自己的独立平台Oak（橡树），并且用此平台开发的项目竞标，可惜失败了。

这时候世界上最早的软件公司诞生了：网景（Netscape），受到了当时浏览器的启发，SUN公司工程师开发出了HotJava浏览器。1995年5月23日推出了Java语言及JDK1.0开发包，1996年才开放下载与使用。1997年传到中国。

SUN公司是一家硬件开发技术公司，最早代表产品是小型机（应用于amazon上），其操作系统@#$%是一种类Unix操作系统。最初的电子商务（第一代）是由IBM提出来的。到90年代末因为经济发展问题产生了互联网低潮，对行业来说是一个严重的伤害。SUN公司在低潮后没有回复往日的经济实力。但是在Oracle公司收购之前，最想收购他的是IBM，其产品线上的技术基础就是Java语言（通过Java赚到最多钱的是IBM）。不管公司如何，Java依然是一个最稳定，最可靠的编程语言，可以承担大型服务器的程序开发任务。

从最初到现在Java也出现了一系列技术发展：

  • Java标准开发（J2SE，JAVA SE）：提供程序底层支持，实现桌面程序开发；
  • Java嵌入式开发（J2ME，JAVA ME）：如GREEN项目等消费电子产品的软件开发，后来被Android取代。再后来版权问题，Google公司推出了自己语言Kotlin进行安卓开发。
  • Java企业开发（J2EE，JAVA EE）：主要是进行企业平台搭建，现在主要是互联网平台，处理大用户量高并发访问的情况。

1.2 Java语言特点

Java之所以可以得到持续的发展力和良好的生态系统，主要取决于Java自身的技术特征：

  1. Java本身是半开源的产品，厂商得以接触到Java底层，使得Java开发更加透明（对比微软.NET）。
  2. 面向对象的编程语言，使Java语法结构更加方便开发人员理解，并且面向对象的设计思想还在进行着扩充（不同JDK版本）。
  3. 提供方便的内存回收机制，对于一些编程语言里面需要明确的手工对对象进行回收释放，否则程序将无法正常提供支持，而Java可以提供自动的内存回收操作，使得内存处理会更方便一些。
  4. 避免了复杂的内存指针问题，使用引用来替代指针；指针虽然是高效的处理方式，但需要较强的逻辑分析。在Java中直接使用引用就可以简化指针的处理，而引用也是在初学过程之中最难理解的部分。
  5. Java是为数不多支持多线程开发的编程语言，可以使得在单位时间内处理的性能得到提升。而多线程也是Java开发最为难以理解的部分，正确的多线程处理才是提升处理能力的核心所在。
  6. Java提供有最高效的网络处理能力，可以基于NIO实现更高效的数据的传输处理。
  7. Java具有良好的可移植性，这样可以提升程序应用范围。
  8. Java足够简单。

1.3 Java可移植性

Java语言之中最大的特点在于其可移植性的支持，所谓的可移植性是指统一程序可在不同操作系统中部署，减少了开发难度，在Java里面实现可移植性控制，在Java里如果想要实现可移植性的控制，需要依靠JVM（Java虚拟机）。它是由软件或硬件模拟出来的计算机，所有程序只要有JVM的支持就可以实现程序的执行，不同操作系统上会有不同版本的JVM，实现可移植性。

计算机编程语言：编译型、解释型。Java两者都有，编译：javac.exe，解释：java.exe。所有程序解释都需要在Java虚拟机上进行处理。编译器生成字节码文件，它是一种通用式的程序，JVM读取并处理编译过的字节码文件*.class，将程序在特定平台上运行。所有程序并不是直接在操作系统上运行，而是通过系统中的JVM执行，这样以来肯定要比直接在系统上执行慢，但由于硬件技术的飞速发展，这些问题亦可以忽略，但依然存在着JVM调优问题，发挥出操作系统的全部性能。

1.4 JDK介绍与安装

Java属于编译型与解释型开发语言，进行Java开发要JDK（Java Development Kit，Java开发工具包）的安装配置。JDK去官网获得，需要在本地配置。

JDK标志性历史版本：

  • 1995.05.23 JDK 1.0 开发包发布，1996年正式提供下载。
  • 1998.12.04 JDK 1.2 版本推出，Java正式更名为Java2（Java升级版）
  • 2005.05.23 JDK 1.5 在Java十周年大会上推出，带来了新特性，决定了Java之后10年的技术核心。
  • 2014年 JDK1.8版本支持Lamda表达式，可以使用函数式式编程。
  • 2017年 JDK1.9版本提升了1.8稳定性。
  • 2018年 JDK1.10版本 属于1.9的稳定版。

如果进行实际项目生产环境部署，现阶段以JDK1.8为主。1.9和1.10差别不大。JRE指Java运行环境，只提供解释功能，不具备开发功能。
安装：Windows平台直接1 运行安装JDK，2 通用JRE，3 环境变量添加javac.exe（路径间分号分割）

1.5 HelloWorld

几乎所有语音第一个程序都是“Hello World!!”，因为最初C语言的第一个程序就是如此。

String args[] 和 String []args均可

Java程序是需要对源代码处理两次后才能正常执行的：

  • 对代码程序进行编译：javac HelloWorld.java，会产生一个HelloWorld.class的字节码文件。利用JVM进行编译，可得到一套与平台无关的字节码文件，*.class
  • 在JVM上进行程序的解释：java HelloWorld，解释的就是字节码文件，执行时不需要加后缀。
1. 类的定义

Java程序开发中最基础的单元是类，所有的程序都必须封装在类中执行，类的基本定义语法如下：[public] class 类名称 {}[]指可选部分。

在本程序中定义类为HelloWorld，而类的定义有两种：

  • public class 类名称 {}

  • class 类名称 {}

  • 用public class 类名称 {}定义类时类名称必须要与文件名一致，一个文件中只 允许存在一个此种定义。

  • 用class 类名称 {}定义类时类名称可以与文件名不一致，但是编译后的*.class文件名与class类名称一致。在一个*.java文件中可以有多个class定义，编译后形成多个*.class文件。

在以后项目开发中，很少有一个*.java源代码中定义多个类的情况，对于今后的开发而言，一个*.java里面定义一个public class类就够了。
Java语言有明确的命名要求，定义类名时要求每个单词首字母大写，如HelloWorld、TestDemo等。包名小写。

2. 主方法

public static void main(String args[]){}

主方法是所有程序执行的起点，并一定要定义在类之中。主方法所在的类被称为“主类”，所有主类都将采用public class 类名称 {}来定义。称作程序起点，客户端操作。

3. 屏幕打印

System.out.println();打印并换行,System.out.print();打印不换行

1.6 JShell交互式编程工具

Shell是脚本程序的含义，在很多的编程语言里面，为了方便编程人员的开发，都会提供Shell交互式编程环境（JDK 1.9版本）。有时可能只是为了进行一些简短的程序验证，但是在Java中要编写很多结构代码才能实现，为了方便这种应用情景，Java提供了JShell的指令，直接运行即可。

除了直接在JShell命令之中进行程序编写之外，也可将一些内容直接交由一些文件进行保存。
示例：定义一个程序文件，希望在JShell里直接执行程序。

使用JShell只需要编写核心结构的代码即可，减少了对于结构化的需求。迫于其他语言的压力（Python等都有），添加的JShell，但实用性并不高。

1.7 CLASSPATH环境属性

Classpath完整解释需要很多知识，这里只对其做先期介绍。

假设在“d:\”下提供有一个Hello.class的字节码文件，在这样情况下可在次目录下可直接用java命令进行Hello.class的解释。但是如果脱离此目录，将当前目录修改为“C:\”（没有Hello.class），如果再次进行程序解释，会出现以下提示信息：

错误: 找不到或无法加载主类 HelloWorld
原因: java.lang.ClassNotFoundException: HelloWorld

是因为当前目录没有字节码文件，现在的需求是可以在不同目录中都执行“d:\”下的HelloWorld.class文件，因此在这样处理环境下只能依靠CLASSPATH环境属性来完成。

定义CLASSPATH环境属性：

SET CLASSPATH=d:\

当设置了CLASSPATH之后，这时在Java程序解释的时候会自动的通过CLASSPATH设置路径进行类的加载，是因为JVM解释程序时候需要CLASSPATH的支持。但是默认情况下所有解释的类都默认当在前目录加载的。很明显到处设置CLASSPATH就会造成系统的混乱，对于CLASSPATH还是应该采取默认设置方式，所以这个时候要想只通过当前目录加载，可将CLASSPATH设为“.”

从当前目录加载类：

SET CLASSPATH=.

在安装其他与Java相关的开发软件的时候，他有可能会自动修改默认的CLASSPATH值，这种情况下就需要手动设置回来。

但是需要注意，CLASSPATH是在一个命令行下的配置，如果关闭该命令行，那么相关的属性配置将消失。所以最好的办法是将其定义为全局属性，则可以直接在系统中追加有一个属性信息：在环境变量中添加“CLASSPATH=.”

PATH和CLASSPATH区别：

  • PATH：是操作系统提供的路径配置，定义所有可执行程序的路径。
  • CLASSPATH：是由JRE提供，用于定义Java程序解释时的加载路径，默认设置为在当前目录加载。可通过“SET CLASSPATH=路径”的命令来进行定义。
    JVM→CLASSPATH定义的路径→加载 *.class文件
